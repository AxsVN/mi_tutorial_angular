Funcionamiento de Angular

Angular es un framework basado en componentes que organiza la aplicación en unidades modulares y reutilizables. A continuación, se describe cómo funciona Angular y cómo actúa sobre las diferentes partes de la aplicación:


1. Angular actúa sobre componentes, no sobre páginas

Angular organiza la aplicación en componentes. Cada componente tiene tres partes principales:

  • HTML: La plantilla que define la estructura de la interfaz de usuario.
  • TypeScript: Contiene la lógica y los métodos asociados al componente.
  • CSS/SCSS: Los estilos que se aplican solo a ese componente.

Esto permite dividir la aplicación en partes más pequeñas y reutilizables, facilitando el mantenimiento y la escalabilidad.


2. Modificación de ficheros

En Angular, los ficheros que se modifican son principalmente los siguientes:

  • HTML: Aquí se define la estructura y el contenido visible de la interfaz de usuario.
  • TypeScript: El archivo de lógica donde se manejan las interacciones, los eventos y los datos.
  • CSS/SCSS: El archivo que contiene los estilos específicos de cada componente.

Angular permite una gran flexibilidad y organización, con la ventaja de que los estilos de un componente solo afectan a ese componente en particular.


3. Enrutamiento

Angular maneja la navegación entre diferentes vistas (o componentes) a través del sistema de enrutamiento, lo que permite cambiar de componente sin recargar la página. Esto mejora el rendimiento y la experiencia del usuario.

Las rutas se configuran en el archivo app-routing.module.ts, y permiten asignar URLs específicas a cada componente de la aplicación.


4. Referencia entre ficheros

Angular organiza los ficheros y los componentes de forma modular. Los ficheros se referencian entre sí de las siguientes maneras:

  • Importación de componentes: Si un componente necesita usar otro, se importa en el archivo TypeScript correspondiente. Por ejemplo:
    • import { ComponenteB } from '../componente-b/componente-b.component';

    Y luego se incluye en la plantilla HTML:

    <app-componente-b></app-componente-b>
  • Servicios: Los servicios se definen en archivos .ts y se inyectan en los componentes que los necesitan para compartir lógica o datos. Ejemplo de inyección de servicio:
    • constructor(private miServicio: MiServicio) {}
  • Módulos: Los módulos agrupan componentes y servicios relacionados. Pueden ser importados en otros módulos. Ejemplo de declaración de un módulo:
    • @NgModule({
	  declarations: [ComponenteA, ComponenteB],
	  imports: [CommonModule],
	  exports: [ComponenteA, ComponenteB]
	})
	export class ModuloCompartido {}

Resumiendo lo comentado...

Angular organiza la aplicación en componentes modulares que gestionan su propia lógica y vista. Los ficheros HTML, TypeScript y CSS se modifican para cambiar la interfaz, la lógica o los estilos. Angular también utiliza el enrutamiento para manejar la navegación entre componentes sin recargar la página, y permite la referencia entre componentes, servicios y módulos para crear aplicaciones eficientes y escalables.
